# Наследование в классах

### Содержание проекта:
В пакете `src/` находятся классы `Product` и `Category`, для которых определены следующие свойства:
1. Для класса `Product`
- название (name),
- описание (description),
- цена (price),
- количество в наличии (quantity).
2. Для класса `Category` 
- название (name),
- описание (description),
- список товаров категории (products).

Для класса `Category` также добавлены два атрибута класса: количество категорий и количество товаров.
Для обоих классов определены магические методы `__str__`, `__add__`, `__iter__` и `__next__`.

Также в модуле `main.py` реализовано дополнительное задание: функция `load_categories_from_json` подгружает
данные по категориям и товарам, а также создает объекты классов из JSON-файла, который находится в папке `data/`. 

### В домашней работе 16.1:
1. Созданы два класса наследников класса Product:
- «Смартфон» (Smartphone) и «Трава газонная» (LawnGrass).
Класс Smartphone расширен атрибутами: производительность (efficiency), модель (model), 
объем встроенной памяти (memory), цвет (color).
Класс LawnGrass расширен атрибутами: страна-производитель (country), срок прорастания 
(germination_period), цвет (color).
2. Ограничения сложения
- Доработана функциональность сложения таким образом, чтобы можно было складывать товары
только из одинаковых классов продуктов.
- Новая функциональность не дает возможности сложить объекты разных классов, например смартфон и траву газонную. 
При попытке сложения таких экземпляров выбрасывается ошибка `TypeError`.
- Для реализации ограничений используется функция `type()`.
3. Ограничения добавления продукта
- Доработан метод, который добавляет продукт в категорию таким образом, чтобы не было 
возможности добавить вместо продукта или его наследников любой другой объект.
- Для реализации ограничений используются функции `issubclass()` и `isinstance()`.
4. Написаны тесты для новой функциональности.

Для всех модулей проекта написаны тесты.
